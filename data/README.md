# Data Directory

This directory contains the application's data files, including the main database, backups, and email archives.

## Structure

```
data/
├── reit_newsletter.db          # Main SQLite database (PRODUCTION)
├── reitsheet.db               # Legacy/test database (investigate if needed)
├── backups/                   # Database backups
│   ├── reit_newsletter_*.db   # Recent backups (keep last 5)
│   └── archived/              # Old backups (30+ days)
├── newsletters/               # Generated newsletter HTML files (.gitignored)
├── s3-emails/                 # Email archives from S3 (.gitignored)
├── test-emails/               # Sample emails for testing (.gitignored)
└── *.json                     # Analysis output files (.gitignored)
```

## Database Files

### reit_newsletter.db
**Main production database** containing:
- Companies table
- Press releases
- Newsletters
- Review emails
- Relevance decisions

### reitsheet.db
**Legacy database** - appears to be duplicate or old version.

## Backup Policy

- **Recent backups**: Keep last 5 in `backups/` folder
- **Archived backups**: Everything older than 1 week in `backups/archived/`
- **Rotation**: Automated via scripts (run daily)

## Email Archives

### s3-emails/
Production email archives synchronized from AWS S3.
**Not tracked in git** - large files, local only.

### test-emails/
Sample emails used for parser testing and development.
**Not tracked in git** - test data only.

## Generated Files

### newsletters/
HTML newsletter files generated by the application.
**Not tracked in git** - regenerated as needed.

### *.json
Analysis output files (e.g., `pr_email_parser_results.json`).
**Not tracked in git** - generated from database data.

## Git Tracking

**Tracked:**
- reit_newsletter.db (main database)
- reitsheet.db (if needed)
- This README

**Not Tracked (.gitignored):**
- backups/
- newsletters/
- s3-emails/
- test-emails/
- *.json files
- *.html files

## Maintenance

- Run backup cleanup weekly: `scripts/cleanup_old_backups.sh`
- Check database integrity: `sqlite3 reit_newsletter.db "PRAGMA integrity_check;"`
- View database size: `du -h reit_newsletter.db`
